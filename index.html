你的公网 IP 与地理位置说明

公网 IP 是你在互联网上的对外地址，网站和 App 都能看到它，并据此推断你的大致地理位置、网络服务商与时区。上方位置来自对你公网 IP 的地理库查询，属城市级估算，未必精确到街道。若你已连 VPN，这里应显示你所选节点所在地，而不是你的真实城市。

什么是 DNS 泄漏？为什么纯网页测不全

DNS 把域名解析成 IP。若你连了 VPN，DNS 查询却仍走本地 ISP 的解析器，你访问了哪些网站就会暴露给 ISP，这就是 DNS 泄漏。需诚实说明：真正的 DNS 泄漏检测要服务器侧配合（让设备解析一个唯一子域、再回看是哪台解析器查的），纯浏览器端无法完整测出。务实做法是连上 VPN 后到专门的 DNS 泄漏检测网站实测一次。

什么是 WebRTC 泄漏？它如何绕过 VPN

WebRTC 是浏览器内置的实时通信能力。网页 JS 可创建 RTCPeerConnection，浏览器为打通 P2P 会通过 ICE 收集你的各种网络地址（本地局域网 IP、以及经 STUN 服务器反射得到的公网 IP），网页能读取这些候选——这就是 WebRTC 泄漏，全程静默、无权限弹窗。自 2020 年起 Chrome 与 Edge 用 mDNS 把本地 IP 换成随机 .local 主机名来遮蔽，但公网 IP 仍可能经 STUN 反射暴露，即使你挂着 VPN。上方检测就是实测你当前浏览器会暴露哪些地址。

怎么读这些结果：连 VPN 后该看到什么

理想状态：公网 IP 与地理位置显示为你所选 VPN 节点所在地、WebRTC 不暴露你的真实公网 IP、浏览器时区也与节点地区一致。若公网 IP 仍是你的真实位置，或 WebRTC 暴露了真实公网 IP，说明流量没真正走 VPN 或存在泄漏——可尝试换节点、换协议后重测。还没安装、想换一个连接更稳的客户端，可前往 LetsVPN 官方下载页。

FAQ

用 VPN 之后我的真实 IP 还会被网站看到吗？会 DNS 泄漏吗？

正常情况下，连上 VPN 后网站看到的是节点 IP 而非你的真实 IP。但要诚实提醒：有独立评测指出快连存在 DNS 泄漏风险、且切换服务器后 IP 未必成功更换——这属单一且时间较早的社区评测，未经官方或第二来源证实。若你很在意隐私，务实做法是连上后用本页这类 IP／DNS 泄漏检测实测一次，确认真实 IP 与 DNS 没外露；若发现泄漏，可换节点、换协议再验证。

连上 VPN 后 IP 显示在哪？真的有隐藏我真实位置吗？

连上后你对外显示的是所选节点所在地的 IP，例如选香港节点就显示香港位置，这是 VPN 隐藏真实位置的基本原理。你可以在客户端切换不同地区节点来改变对外 IP。要提醒的是：有独立评测反映快连切服务器后 IP 未必都成功更换、且存在 DNS 泄漏风险（单一较早期评测、未经多源证实），所以连上后建议自行用 IP 检测确认位置是否真的改变。

VPN 连上后 IP 没变、查询还是本地，是没连成功吗？

是的，IP 没变通常代表流量没真正走 VPN。社区反映 LetsVPN 切换服务器后 IP 未必更换成功，这正是这类情况。处理方式：先换一条不同地区的节点重连、再换协议试试；清缓存后重新登录、更新到最新版或重装；确认本机没有其他 VPN/代理在抢流量。重连后再用本页查一次 IP，显示成你选的国家才算成功。安卓也要确认没被系统杀后台。

什么是 WebRTC 泄漏？为什么连了 VPN 还可能暴露真实 IP？

WebRTC 是浏览器内置的点对点通信能力。网页只要用几行 JS 创建 RTCPeerConnection，浏览器就会通过 ICE 收集网络地址，其中经 STUN 服务器反射得到的公网 IP 可能就是你的真实 IP，即使你挂着 VPN，网页也能静默读取、无任何权限弹窗。自 2020 年起 Chrome/Edge 以 mDNS 遮蔽本地 IP，但公网 IP 仍可能漏。用本页上方的 WebRTC 检测即可看到你当前浏览器会暴露哪些地址。

老实说，现有公开素材没有快连隐私政策原文或可验证的「不留记录（no-log）」内容，无法替它背书，需以官方政策原文与第三方审计确认。客观参考：有独立评测对它提出 DNS 泄漏疑虑（单源、较早、未经多源证实），这与「严格不留记录」相左。务实判断：把它当切区／一般用途工具，高敏感操作另选有公开审计报告的服务，并连上后自行做泄漏检测验证实际表现，而非只信宣传语。
