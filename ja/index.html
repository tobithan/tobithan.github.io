あなたの公開 IP と位置情報について

公開 IP はインターネット上での外向きアドレスで、サイトやアプリから見え、そこからおおよその位置・ISP・タイムゾーンが推定されます。上の位置は公開 IP のジオ照会によるもので、都市レベルの推定であり番地まで正確ではありません。VPN 接続中なら、実際の都市ではなく選んだノードの所在地が表示されるはずです。

DNS 漏れとは？ なぜブラウザだけでは完全に測れないのか

DNS はドメイン名を IP に変換します。VPN 接続中でも DNS 問い合わせがローカル ISP のリゾルバを通ると、訪問先が ISP に露出します。これが DNS 漏れです。正直に言うと、本当の DNS 漏れ検査にはサーバー側の仕組み（端末が一意のサブドメインを解決し、どのリゾルバが問い合わせたかを見る）が必要で、ブラウザ単体では完全には測れません。接続後に専用の DNS 漏れチェックサイトで一度実測するのが現実的です。

WebRTC 漏れとは？ どうやって VPN を迂回するのか

WebRTC はブラウザ内蔵のリアルタイム通信機能です。ページは JavaScript で RTCPeerConnection を作成でき、ブラウザは P2P 確立のため ICE で各種ネットワークアドレス（ローカル LAN の IP、STUN サーバーで反射した公開 IP）を収集し、ページはその候補を読み取れます。これが WebRTC 漏れで、許可ダイアログもなく静かに起こります。2020 年以降 Chrome と Edge は mDNS でローカル IP をランダムな .local ホスト名に隠しますが、公開 IP は VPN 接続中でも STUN 反射で漏れ得ます。上の検査は、今のブラウザがどのアドレスを露出するかを実測します。

結果の読み方：VPN 接続時に見えるべき状態

理想は、公開 IP と位置が選んだ VPN ノードの所在地になり、WebRTC が実 IP を露出せず、ブラウザのタイムゾーンもノードの地域と一致している状態です。公開 IP が実際の位置のままだったり、WebRTC が実 IP を露出する場合は、通信が VPN を通っていないか漏れがあります。ノードやプロトコルを変えて再検査しましょう。未導入で、より安定して接続できるクライアントを探しているなら LetsVPN 公式ダウンロードページ へ。

FAQ

VPN 接続後も実 IP はサイトに見える？ DNS は漏れる？

通常、VPN 接続後はサイトにはノード IP が見え、実 IP は見えません。ただ正直に言うと、ある独立レビューが LetsVPN の DNS 漏れリスクと、サーバー切替後に IP が変わらないことがある点を指摘しました。これは単一かつ古めのコミュニティ評価で、公式や第二の情報源では確認されていません。プライバシーを重視するなら、接続後に本ページのような IP/DNS 漏れチェックで実 IP と DNS が露出していないか確認し、漏れていればノードやプロトコルを変えて再検証するのが現実的です。

接続後 IP はどこに表示される？ 本当に位置を隠せる？

接続後の外向き IP は選んだノードの所在地のものになります。香港ノードなら香港の位置が表示され、これが VPN で実際の位置を隠す原理です。クライアントで地域ノードを切り替えれば外向き IP も変わります。注意点として、独立レビューはサーバー切替後に IP が変わらないことがある点と DNS 漏れリスク（単一・古め・多源未確認）を報告しているため、接続後は IP 検査で位置が実際に変わったか確認するのが安全です。

接続後も IP が変わらず、ローカルのまま。失敗？

はい、IP が変わらないのは通常、通信が VPN を通っていないことを意味します。LetsVPN はサーバー切替後に IP が変わらないことがあるとコミュニティで報告されており、まさにこのケースです。対処：別地域のノードで再接続し、プロトコルも変更；キャッシュを消して再ログイン、最新版に更新または再インストール；他の VPN/プロキシが通信を奪っていないか確認。再接続後に本ページで IP を再確認し、選んだ国が表示されて初めて成功です。Android では OS にバックグラウンドで終了させられていないかも確認を。

WebRTC 漏れとは？ VPN を使っても実 IP が漏れるのはなぜ？

WebRTC はブラウザ内蔵の P2P 機能です。数行の JavaScript でページが RTCPeerConnection を作ると、ブラウザは ICE でネットワークアドレスを収集し、STUN サーバーで反射した公開 IP が VPN 接続中でも実 IP になり得て、許可ダイアログもなく静かにページに読み取られます。2020 年以降 Chrome/Edge は mDNS でローカル IP を隠しますが、公開 IP は依然漏れ得ます。上の WebRTC 検査で、今のブラウザがどのアドレスを露出するか確認できます。

LetsVPN のプライバシーポリシーは？ 本当にノーログ？

正直なところ、公開素材には LetsVPN のポリシー原文や検証可能なノーログの根拠がなく、保証はできません。公式ポリシー原文と第三者監査での確認が必要です。参考までに、独立レビューは DNS 漏れの懸念（単一・古め・多源未確認）を挙げており、厳格なノーログとは相容れません。現実的には、地域切替や一般用途のツールと位置づけ、機微な操作は監査済みサービスを使い、接続後は自分で漏れチェックを行って実際の挙動を確認するのが賢明です。
