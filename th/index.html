เครื่องมือฝั่งไคลเอ็นต์อย่างแท้จริง: เครื่องมือจะอ่านและแสดง IP สาธารณะที่คุณเปิดเผยทางอินเทอร์เน็ตในเบราว์เซอร์ของคุณโดยตรง ตำแหน่งทางภูมิศาสตร์และเขตเวลาของคุณ รวมถึง WebRTC ของเบราว์เซอร์ของคุณจะทำให้ IP ท้องถิ่นหรือ IP จริงของคุณรั่วไหลไปยังหน้าเว็บหรือไม่ การตรวจสอบทุกครั้งจะทำงานภายในเครื่อง โดยไม่มีการอัปโหลดอะไรเลย หลังจากเชื่อมต่อ VPN เช่น LetsVPN แล้ว ให้รันอีกครั้งเพื่อยืนยันว่าไซต์เห็น IP ของโหนด ไม่ใช่ IP จริงของคุณ

เกี่ยวกับ IP สาธารณะและตำแหน่งของคุณ

IP สาธารณะของคุณคือที่อยู่ภายนอกของคุณบนอินเทอร์เน็ต เว็บไซต์และแอปสามารถดูและอนุมานตำแหน่งโดยประมาณ, ISP และเขตเวลาของคุณได้ ตำแหน่งด้านบนมาจากการค้นหาทางภูมิศาสตร์ของ IP สาธารณะของคุณ ดังนั้นจึงเป็นการประมาณการระดับเมือง ไม่ใช่ที่อยู่ หากคุณใช้ VPN ควรแสดงตำแหน่งของโหนดที่คุณเลือก ไม่ใช่เมืองจริงของคุณ

DNS รั่วไหลคืออะไร และเหตุใดเบราว์เซอร์เพียงอย่างเดียวจึงไม่สามารถทดสอบได้อย่างสมบูรณ์

DNS แก้ไขชื่อโดเมนเป็น IP หากคุณใช้ VPN แต่การสืบค้น DNS ยังคงผ่านตัวแก้ไข ISP ในพื้นที่ของคุณ เว็บไซต์ที่คุณเยี่ยมชมจะถูกเปิดเผยต่อ ISP ของคุณ นั่นคือการรั่วไหลของ DNS พูดตามตรง: การทดสอบการรั่วไหลของ DNS ที่แท้จริงจำเป็นต้องมีส่วนประกอบฝั่งเซิร์ฟเวอร์ (อุปกรณ์ของคุณแก้ไขโดเมนย่อยที่ไม่ซ้ำกันและการทดสอบจะดูว่าตัวแก้ไขตัวใดถาม) ซึ่งเบราว์เซอร์เพียงอย่างเดียวไม่สามารถทำได้เต็มที่ แนวทางปฏิบัติคือการเรียกใช้ไซต์ทดสอบการรั่วไหลของ DNS โดยเฉพาะเมื่อคุณเชื่อมต่อแล้ว

WebRTC รั่วไหลคืออะไร และจะข้าม VPN ได้อย่างไร

WebRTC คือ API การสื่อสารแบบเรียลไทม์ในตัวของเบราว์เซอร์ หน้าเว็บสามารถสร้าง RTCPeerConnection ใน JavaScrIPt; ในการตั้งค่าเพียร์ทูเพียร์ เบราว์เซอร์จะรวบรวมที่อยู่เครือข่ายของคุณผ่าน ICE รวมถึง LAN IP ในพื้นที่ของคุณและ IP สาธารณะที่แสดงโดยเซิร์ฟเวอร์ STUN และเพจสามารถอ่านผู้สมัครเหล่านั้นได้ นั่นคือการรั่วไหลของ WebRTC และเกิดขึ้นอย่างเงียบๆ โดยไม่มีการแจ้งเตือน ตั้งแต่ปี 2020 Chrome และ Edge ใช้ MDNS เพื่อปกปิด IP ภายในเครื่องที่อยู่เบื้องหลังชื่อโฮสต์ .local แบบสุ่ม แต่ IP สาธารณะยังคงรั่วไหลผ่านตัวเลือกที่สะท้อนกลับของ STUN ได้ แม้ว่า VPN จะเปิดอยู่ก็ตาม การทดสอบข้างต้นแสดงให้เห็นว่าที่อยู่ใดที่เบราว์เซอร์ของคุณเปิดเผยอยู่ในปัจจุบัน

วิธีอ่านผลลัพธ์เหล่านี้: สิ่งที่คุณควรเห็นบน VPN

สถานะในอุดมคติ: IP สาธารณะและตำแหน่งจะแสดงตำแหน่งของโหนด VPN ที่คุณเลือก, WebRTC จะไม่เปิดเผย IP สาธารณะจริงของคุณ และเขตเวลาของเบราว์เซอร์ก็ตรงกับภูมิภาคของโหนดเช่นกัน หาก IP สาธารณะยังคงเป็นตำแหน่งจริงของคุณ หรือ WebRTC เปิดเผย IP สาธารณะจริงของคุณ การรับส่งข้อมูลจะไม่ผ่าน VPN หรือมีการรั่วไหล ลองใช้โหนดหรือโปรโตคอลอื่นแล้วทดสอบอีกครั้ง หากคุณยังไม่ได้ติดตั้งและต้องการไคลเอนต์ที่มีการเชื่อมต่อที่เสถียรยิ่งขึ้น ให้ไปที่ หน้าดาวน์โหลด LetsVPN อย่างเป็นทางการ

  • ★ ภายในปี 2568 มีผู้ใช้ VPN ประมาณ 1.75 พันล้านคน หรือประมาณหนึ่งในสามของผู้ใช้อินเทอร์เน็ตทั้งหมด

  • เบราว์เซอร์เดสก์ท็อปทั่วไป (Chrome / Edge / Firefox / Safari) ทั้งหมดเปิดใช้งาน WebRTC ตามค่าเริ่มต้น จากนี้ เราประเมินว่าในปี 2026 ผู้ใช้ VPN จำนวนมากที่ไม่ได้ปิดการใช้งาน WebRTC ยังคงมีความเสี่ยงต่อการรั่วไหลของ IP สาธารณะผ่านการสะท้อน STUN (ช่วงโดยประมาณ ไม่ได้วัด)

พื้นฐาน: ★ รายการอ้างอิงสถิติสาธารณะ (ที่มาด้านล่าง); ส่วนที่เหลือเป็นการประมาณการของไซต์นี้จากข้อมูลสาธารณะ บวกกับข้อเท็จจริงที่ว่า WebRTC เปิดอยู่โดยค่าเริ่มต้น โดยมีป้ายกำกับว่า "ประมาณการ ไม่ใช่บุคคลที่หนึ่ง" และไม่แสดงเป็นข้อมูลที่วัดได้

FAQ

หลังจากเชื่อมต่อ VPN แล้ว เว็บไซต์ยังสามารถเห็น IP จริงของฉันได้หรือไม่ DNS สามารถรั่วไหลได้หรือไม่?

โดยปกติ เมื่อเชื่อมต่อ VPN แล้ว ไซต์จะเห็น IP ของโหนด ไม่ใช่ IP จริงของคุณ แต่พูดตามตรง: การตรวจสอบอิสระทำเครื่องหมายความเสี่ยงการรั่วไหลของ DNS บน LetsVPN และ IP ไม่ได้เปลี่ยนแปลงเสมอไปหลังจากเปลี่ยนเซิร์ฟเวอร์ - นั่นคือการตรวจสอบชุมชนเก่าเพียงครั้งเดียว ไม่ได้รับการยืนยันจากผู้ขายหรือแหล่งที่มาที่สอง หากความเป็นส่วนตัวมีความสำคัญต่อคุณ การดำเนินการในทางปฏิบัติคือการทดสอบการรั่วไหลของ IP/DNS เช่นนี้หลังจากเชื่อมต่อและยืนยันว่า IP และ DNS จริงของคุณไม่ถูกเปิดเผย หากเป็นเช่นนั้น ให้สลับโหนดหรือโปรโตคอลและตรวจสอบอีกครั้ง

หลังจากเชื่อมต่อแล้ว IP ของฉันจะแสดงที่ไหน? มันซ่อนตำแหน่งของฉันจริงๆเหรอ?

เมื่อเชื่อมต่อแล้ว IP ภายนอกของคุณจะเป็นตำแหน่งโหนดที่คุณเลือก - เลือกโหนดฮ่องกงแล้วคุณแสดงตำแหน่งของฮ่องกง นั่นคือวิธีที่ VPN ซ่อนตำแหน่งจริงของคุณ คุณสามารถสลับโหนดภูมิภาคในไคลเอนต์เพื่อเปลี่ยน IP ภายนอกของคุณได้ โปรดทราบว่าการตรวจสอบโดยหน่วยงานอิสระรายงานว่า IP ไม่ได้เปลี่ยนแปลงเสมอไปหลังจากเปลี่ยนเซิร์ฟเวอร์และพบว่ามีความเสี่ยงการรั่วไหลของ DNS (การตรวจสอบแบบเก่าและแบบเดี่ยว ไม่ใช่การยืนยันจากหลายแหล่ง) ดังนั้นหลังจากเชื่อมต่อแล้ว ก็ควรตรวจสอบด้วยการทดสอบ IP ว่าตำแหน่งของคุณเปลี่ยนแปลงจริง ๆ

หลังจากเชื่อมต่อแล้ว IP ของฉันไม่เปลี่ยนแปลงและการค้นหายังคงแสดงอยู่ในเครื่อง - ล้มเหลวหรือไม่

ใช่ IP ที่ไม่เปลี่ยนแปลงมักจะหมายความว่าการรับส่งข้อมูลไม่ได้ผ่าน VPN จริงๆ ชุมชนรายงานว่า IP ของ LetsVPN ไม่ได้เปลี่ยนแปลงเสมอไปหลังจากเปลี่ยนเซิร์ฟเวอร์ ซึ่งก็คือกรณีนี้นั่นเอง การแก้ไข: เชื่อมต่อใหม่กับโหนดภูมิภาคอื่น จากนั้นลองใช้โปรโตคอลอื่น ล้างแคชแล้วเข้าสู่ระบบอีกครั้ง อัปเดตเป็นเวอร์ชันล่าสุดหรือติดตั้งใหม่ ตรวจสอบให้แน่ใจว่าไม่มี VPN/พร็อกซีอื่นรับส่งข้อมูล หลังจากเชื่อมต่อใหม่แล้ว ให้ตรวจสอบ IP ของคุณอีกครั้งที่นี่ - จะนับว่าสำเร็จก็ต่อเมื่อแสดงประเทศที่คุณเลือกเท่านั้น บน Android ตรวจสอบให้แน่ใจว่าระบบไม่ได้ปิดการทำงานในเบื้องหลัง

WebRTC รั่วไหลคืออะไร และเหตุใด IP จริงของฉันจึงยังรั่วไหลบน VPN ได้

WebRTC เป็น API แบบเพียร์ทูเพียร์ในตัวของเบราว์เซอร์ ด้วย JavaScrIPt เพียงไม่กี่บรรทัด เพจก็สามารถสร้าง RTCPeerConnection และเบราว์เซอร์จะรวบรวมที่อยู่เครือข่ายผ่าน ICE IP สาธารณะที่สะท้อนโดยเซิร์ฟเวอร์ STUN อาจเป็น IP จริงของคุณ - แม้แต่บน VPN - และเพจจะอ่านโดยไม่ต้องแจ้งให้ทราบโดยไม่มีการอนุญาต ตั้งแต่ปี 2020 Chrome/Edge ปิดบัง IP ในเครื่องผ่าน MDNS แต่ IP สาธารณะยังคงรั่วไหลได้ ใช้การทดสอบ WebRTC ด้านบนเพื่อดูว่าเบราว์เซอร์ของคุณเปิดเผยที่อยู่ใดในปัจจุบัน
