Tentang IP publik dan lokasi Anda

IP publik Anda adalah alamat luar Anda di internet. Situs web dan aplikasi dapat melihatnya dan menyimpulkan perkiraan lokasi, ISP, dan zona waktu Anda. Lokasi di atas berasal dari pencarian geografis IP publik Anda, jadi ini merupakan perkiraan tingkat kota, bukan alamat jalan. Jika Anda menggunakan VPN, itu akan menunjukkan lokasi node yang Anda pilih, bukan kota Anda yang sebenarnya.

Apa itu kebocoran DNS, dan mengapa browser saja tidak dapat mengujinya sepenuhnya

DNS menyelesaikan nama domain menjadi IP. Jika Anda menggunakan VPN tetapi permintaan DNS masih melalui penyelesai ISP lokal Anda, situs yang Anda kunjungi akan terekspos ke ISP Anda; itu adalah kebocoran DNS. Sejujurnya: pengujian kebocoran DNS yang sebenarnya memerlukan komponen sisi server (perangkat Anda menyelesaikan subdomain unik dan pengujian tersebut melihat pemecah masalah mana yang diminta), yang tidak dapat dilakukan sepenuhnya oleh browser saja. Pendekatan praktisnya adalah dengan menjalankan situs uji kebocoran DNS khusus setelah Anda terhubung.

Apa itu kebocoran WebRTC, dan bagaimana cara melewati VPN

WebRTC adalah API komunikasi real-time bawaan browser. Halaman web dapat membuat RTCPeerConnection dalam JavaScrIPt; untuk mengatur peer-to-peer, browser mengumpulkan alamat jaringan Anda melalui ICE, termasuk IP LAN lokal Anda dan IP publik yang dICErminkan oleh server STUN, dan halaman tersebut dapat membaca kandidat tersebut. Itu adalah kebocoran WebRTC, dan terjadi secara diam-diam tanpa permintaan izin. Sejak tahun 2020 Chrome dan Edge menggunakan MDNS untuk menutupi IP lokal di balik nama host .local acak, namun IP publik masih dapat bocor melalui kandidat refleksif STUN meskIPun VPN aktif. Tes di atas menunjukkan dengan tepat alamat mana yang saat ini diekspos browser Anda.

Cara membaca hasil ini: apa yang seharusnya Anda lihat di VPN

Keadaan ideal: IP publik dan lokasi menunjukkan lokasi node VPN yang Anda pilih, WebRTC tidak mengekspos IP publik Anda yang sebenarnya, dan zona waktu browser juga cocok dengan wilayah node. Jika IP publik masih merupakan lokasi sebenarnya Anda, atau WebRTC memperlihatkan IP publik Anda yang sebenarnya, lalu lintas tidak benar-benar melalui VPN atau terjadi kebocoran; coba node atau protokol lain dan uji ulang. Jika Anda belum menginstalnya dan menginginkan klien dengan koneksi yang lebih stabil, kunjungi halaman unduh resmi LetsVPN.

FAQ

Setelah menghubungkan VPN, apakah situs masih dapat melihat IP asli saya? Bisakah DNS bocor?

Biasanya, setelah VPN terhubung, situs akan melihat IP node, bukan IP asli Anda. Tapi sejujurnya: tinjauan independen menandai risiko kebocoran DNS di LetsVPN dan bahwa IP tidak selalu berubah setelah berpindah server - itu adalah tinjauan komunitas lama, tidak dikonfirmasi oleh vendor atau sumber kedua. Jika privasi penting bagi Anda, langkah praktisnya adalah menjalankan tes kebocoran IP/DNS seperti ini setelah menghubungkan dan mengonfirmasi bahwa IP dan DNS asli Anda tidak terekspos; jika ya, ganti node atau protokol dan verifikasi lagi.

Setelah terhubung, di mana IP saya muncul? Apakah itu benar-benar menyembunyikan lokasi saya?

Setelah terhubung, IP luar Anda adalah lokasi node yang Anda pilih - pilih node Hong Kong dan Anda menunjukkan lokasi Hong Kong; begitulah cara VPN menyembunyikan lokasi Anda yang sebenarnya. Anda dapat berpindah node wilayah di klien untuk mengubah IP luar Anda. Perhatikan bahwa tinjauan independen melaporkan bahwa IP tidak selalu berubah setelah berpindah server dan melihat risiko kebocoran DNS (ulasan tunggal, lebih lama, tidak dikonfirmasi multi-sumber), jadi setelah menghubungkan, sebaiknya verifikasi dengan tes IP bahwa lokasi Anda benar-benar berubah.

Setelah terhubung, IP saya tidak berubah dan pencarian masih menampilkan lokal - apakah gagal?

Ya, IP yang tidak berubah biasanya berarti lalu lintas sebenarnya tidak melalui VPN. Komunitas telah melaporkan bahwa IP LetsVPN tidak selalu berubah setelah berpindah server, dan inilah kasusnya. Perbaikan: sambungkan kembali ke node wilayah lain, lalu coba protokol lain; bersihkan cache dan masuk lagi, perbarui ke versi terbaru atau instal ulang; pastikan tidak ada VPN/proxy lain yang mengambil lalu lintas. Setelah terhubung kembali, periksa kembali IP Anda di sini - ini dihitung sebagai keberhasilan hanya jika IP tersebut menunjukkan negara pilihan Anda. Di Android, pastikan juga sistem tidak mematikannya di latar belakang.

Apa yang dimaksud dengan kebocoran WebRTC, dan mengapa IP asli saya masih bisa bocor di VPN?

WebRTC adalah API peer-to-peer bawaan browser. Hanya dengan beberapa baris JavaScrIPt, sebuah halaman dapat membuat RTCPeerConnection, dan browser kemudian mengumpulkan alamat jaringan melalui ICE; IP publik yang tercermin oleh server STUN dapat menjadi IP asli Anda - bahkan pada VPN - dan halaman membacanya secara diam-diam tanpa perintah izin. Sejak tahun 2020 Chrome/Edge menutupi IP lokal melalui MDNS, tetapi IP publik masih bisa bocor. Gunakan tes WebRTC di atas untuk melihat alamat mana yang saat ini diekspos browser Anda.

Apa yang tercantum dalam kebijakan privasi LetsVPN - apakah ini benar-benar tanpa pencatatan atau sekadar bicara?

Sejujurnya, materi publik yang tersedia tidak memiliki teks kebijakan privasi LetsVPN atau konten tanpa pencatatan yang dapat diverifikasi, sehingga tidak dapat dijamin; yang memerlukan teks kebijakan resmi dan audit pihak ketiga. Untuk konteksnya, tinjauan independen mengangkat kekhawatiran kebocoran DNS (tunggal, lama, tidak dikonfirmasi multi-sumber), yang bertentangan dengan larangan pencatatan yang ketat. Solusi praktisnya: perlakukan ini sebagai alat peralihan wilayah/penggunaan umum, gunakan layanan yang telah diaudit untuk tugas-tugas yang sangat sensitif, dan jalankan uji kebocoran sendiri setelah tersambung, alih-alih memercayai klaim pemasaran.
