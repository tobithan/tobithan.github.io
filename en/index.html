About your public IP and location

Your public IP is your outward address on the internet. Websites and apps can see it and infer your approximate location, ISP and timezone. The location above comes from a geo lookup of your public IP, so it is a city-level estimate, not a street address. If you are on a VPN, it should show the location of the node you picked, not your real city.

What is a DNS leak, and why a browser alone cannot fully test it

DNS resolves domain names into IPs. If you are on a VPN but DNS queries still go through your local ISP resolver, the sites you visit are exposed to your ISP; that is a DNS leak. To be honest: a true DNS leak test needs a server-side component (your device resolves a unique subdomain and the test sees which resolver asked), which a browser alone cannot fully do. The practical approach is to run a dedicated DNS leak test site once you are connected.

What is a WebRTC leak, and how it bypasses a VPN

WebRTC is the browser's built-in real-time communication API. A web page can create an RTCPeerConnection in JavaScript; to set up peer-to-peer the browser gathers your network addresses via ICE, including your local LAN IP and the public IP reflected by a STUN server, and the page can read those candidates. That is a WebRTC leak, and it happens silently with no permission prompt. Since 2020 Chrome and Edge use mDNS to mask the local IP behind a random .local hostname, but the public IP can still leak through STUN reflexive candidates even while a VPN is on. The test above shows exactly which addresses your browser currently exposes.

How to read these results: what you should see on a VPN

Ideal state: the public IP and location show your chosen VPN node's location, WebRTC does not expose your real public IP, and the browser timezone matches the node region too. If the public IP is still your real location, or WebRTC exposes your real public IP, traffic is not really going through the VPN or there is a leak; try another node or protocol and re-test. FAQ

After connecting a VPN, can sites still see my real IP? Can DNS leak?

Normally, once a VPN is connected sites see the node IP, not your real IP. But to be honest: an independent review flagged DNS leak risk on LetsVPN and that the IP did not always change after switching servers - that was a single, older community review, not confirmed by the vendor or a second source. If privacy matters to you, the practical move is to run an IP/DNS leak test like this one after connecting and confirm your real IP and DNS are not exposed; if they are, switch node or protocol and verify again.

After connecting, where does my IP show? Does it really hide my location?

Once connected, your outward IP is that of the node location you chose - pick a Hong Kong node and you show a Hong Kong location; that is how a VPN hides your real location. You can switch region nodes in the client to change your outward IP. Note that an independent review reported the IP did not always change after switching servers and saw DNS leak risk (single, older review, not multi-source confirmed), so after connecting it is wise to verify with an IP test that your location actually changed.

After connecting, my IP did not change and lookups still show local - did it fail?

Yes, an unchanged IP usually means traffic is not actually going through the VPN. The community has reported that LetsVPN's IP does not always change after switching servers, which is exactly this case. Fixes: reconnect to a different region node, then try another protocol; clear cache and log in again, update to the latest version or reinstall; make sure no other VPN/proxy is taking the traffic. After reconnecting, check your IP again here - it counts as success only when it shows your chosen country. On Android, also make sure the system did not kill it in the background.

What is a WebRTC leak, and why can my real IP still leak on a VPN?

WebRTC is the browser's built-in peer-to-peer API. With just a few lines of JavaScript a page can create an RTCPeerConnection, and the browser then gathers network addresses via ICE; the public IP reflected by a STUN server can be your real IP - even on a VPN - and the page reads it silently with no permission prompt. Since 2020 Chrome/Edge mask the local IP via mDNS, but the public IP can still leak. Use the WebRTC test above to see which addresses your browser currently exposes.

The practical call: treat it as a region-switch / general-use tool, use an audited service for highly sensitive tasks, and run a leak test yourself after connecting rather than trusting marketing claims.
