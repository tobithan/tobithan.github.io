你的公網 IP 與地理位置說明

公網 IP 是你在網際網路上的對外地址，網站和 App 都能看到它，並據此推斷你的大致地理位置、網路服務商與時區。上方位置來自對你公網 IP 的地理庫查詢，屬城市級估算，未必精確到街道。若你已連 VPN，這裡應顯示你所選節點所在地，而不是你的真實城市。

什麼是 DNS 洩漏？為什麼純網頁測不全

DNS 把域名解析成 IP。若你連了 VPN，DNS 查詢卻仍走本地 ISP 的解析器，你訪問了哪些網站就會暴露給 ISP，這就是 DNS 洩漏。需誠實說明：真正的 DNS 洩漏檢測要伺服器側配合（讓裝置解析一個唯一子域、再回看是哪臺解析器查的），純瀏覽器端無法完整測出。務實做法是連上 VPN 後到專門的 DNS 洩漏檢測網站實測一次。

什麼是 WebRTC 洩漏？它如何繞過 VPN

WebRTC 是瀏覽器內建的實時通訊能力。網頁 JS 可建立 RTCPeerConnection，瀏覽器為打通 P2P 會透過 ICE 收集你的各種網路地址（本地區域網 IP、以及經 STUN 伺服器反射得到的公網 IP），網頁能讀取這些候選——這就是 WebRTC 洩漏，全程靜默、無許可權彈窗。自 2020 年起 Chrome 與 Edge 用 MDNS 把本地 IP 換成隨機 .local 主機名來遮蔽，但公網 IP 仍可能經 STUN 反射暴露，即使你掛著 VPN。上方檢測就是實測你當前瀏覽器會暴露哪些地址。

FAQ

用 VPN 之後我的真實 IP 還會被網站看到嗎？會 DNS 洩漏嗎？

正常情況下，連上 VPN 後網站看到的是節點 IP 而非你的真實 IP。但要誠實提醒：有獨立評測指出快連存在 DNS 洩漏風險、且切換伺服器後 IP 未必成功更換——這屬單一且時間較早的社群評測，未經官方或第二來源證實。若你很在意隱私，務實做法是連上後用本頁這類 IP／DNS 洩漏檢測實測一次，確認真實 IP 與 DNS 沒外露；若發現洩漏，可換節點、換協議再驗證。

連上 VPN 後 IP 顯示在哪？真的有隱藏我真實位置嗎？

連上後你對外顯示的是所選節點所在地的 IP，例如選香港節點就顯示香港位置，這是 VPN 隱藏真實位置的基本原理。你可以在客戶端切換不同地區節點來改變對外 IP。要提醒的是：有獨立評測反映快連切伺服器後 IP 未必都成功更換、且存在 DNS 洩漏風險（單一較早期評測、未經多源證實），所以連上後建議自行用 IP 檢測確認位置是否真的改變。

VPN 連上後 IP 沒變、查詢還是本地，是沒連成功嗎？

是的，IP 沒變通常代表流量沒真正走 VPN。社群反映 LetsVPN 切換伺服器後 IP 未必更換成功，這正是這類情況。處理方式：先換一條不同地區的節點重連、再換協議試試；清快取後重新登入、更新到最新版或重灌；確認本機沒有其他 VPN/代理在搶流量。重連後再用本頁查一次 IP，顯示成你選的國家才算成功。安卓也要確認沒被系統殺後臺。

什麼是 WebRTC 洩漏？為什麼連了 VPN 還可能暴露真實 IP？

WebRTC 是瀏覽器內建的點對點通訊能力。網頁只要用幾行 JS 建立 RTCPeerConnection，瀏覽器就會透過 ICE 收集網路地址，其中經 STUN 伺服器反射得到的公網 IP 可能就是你的真實 IP，即使你掛著 VPN，網頁也能靜默讀取、無任何許可權彈窗。自 2020 年起 Chrome/Edge 以 MDNS 遮蔽本地 IP，但公網 IP 仍可能漏。用本頁上方的 WebRTC 檢測即可看到你當前瀏覽器會暴露哪些地址。

老實說，現有公開素材沒有快連隱私政策原文或可驗證的「不留記錄（no-log）」內容，無法替它背書，需以官方政策原文與第三方審計確認。客觀參考：有獨立評測對它提出 DNS 洩漏疑慮（單源、較早、未經多源證實），這與「嚴格不留記錄」相左。務實判斷：把它當切區／一般用途工具，高敏感操作另選有公開審計報告的服務，並連上後自行做洩漏檢測驗證實際表現，而非只信宣傳語。
